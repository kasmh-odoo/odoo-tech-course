Realty Management - Part 1

A Realty management agency has approached your company to implement a property management solution using Odoo. Their requirements are as follows:

Properties

The buildings, apartments, and houses to be managed within the real estate module. Properties include the following attributes based on the current system schema:

  • Name: A descriptive title for the property (Required).
  • Description: A detailed text summary of the property features.
  • Postcode: The postal code where the property is located.
  • Date Availability: The date the property becomes available. Defaults to 3 months from today (Required, No Copy).
  • Expected Price: The initial asking price. Must be greater than 0 (Required).
  • Selling Price: The final price at which the property was sold (Read-only).
  • Property Type: A categorization of the property (e.g., House, Apartment).
  • Bedrooms: Number of bedrooms available. Defaults to 2.
  • Living Area: The interior living space in square meters.
  • Facades: The number of exterior faces the building has.
  • Garage: Indicates if the property includes a garage.
  • Garden: Indicates if the property has a garden.
  • Garden Area: The size of the garden in square meters.
  • Garden Orientation: The direction the garden faces (North, South, East, or West).
  • Total Area: A calculated field summing the living area and garden area.
  • Status: The current lifecycle stage of the property (New, Offer Received, Offer Accepted, Sold, or Cancelled). Defaults to "New".
  • Salesperson: The user responsible for the property. Defaults to the current user.
  • Buyer: The partner who eventually purchases the property.
  • Tags: Custom labels used to categorize or filter properties.

Property Types

The property type of a listing.

  • Name: The type of property (Required).

Property Tags

Additional information about a listing that is relevant for a buyer.

  • Name: An additional piece of information about the property (Required).

Implementation Notes

  • Custom Models: Create appropriate models, views, and access rights for Properties, Tenancies, and Tenants.
  • Inheritance: Tenants are res.partner records. Add fields to the Python model and the existing partner form view.
  • Logic: Tenancies start in draft. Add a button to set to Active only if Date End > Date Start.
  • Automation: Create an ir.cron scheduled action to check daily if a tenancy has reached its end date.
  • Navigation: Create a new top-level menu item titled Realty Management.

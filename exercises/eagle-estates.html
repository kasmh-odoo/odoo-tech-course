Realty Management - Part 1

A Realty management agency has approached your company to implement a property management solution using Odoo. Their requirements are as follows:

Properties

The buildings, apartments, and houses to be managed. Properties have the following required attributes:

  • Name: A name for the property (Required)
  • Construction Date: The date on which the property was constructed (Required)
  • Rent: The base rent price for the property (Required)
  • Rooms: A list of rooms available in the property, including kitchens, bathrooms and storage rooms (Required)
  • Parking Spots: The number of parking spots the property has access to (Required)
  • Surface: The total surface of the property, displayed in meters squared (Required)
  • Furnishing: Whether or not the property is furnished (Required)
  • Address: A reference to the property's address, including: Street, City, Zip Code, State, Country (Required)
  • Available for Rent: Whether the property is available for rent (Required)
  • Is Rented: Whether the property is currently rented. Computed logic: Is there any tenancy active during the current time frame? (Required)

Rooms

The rooms that make up the property.

  • Name: Name of the room (Required)
  • Surface: The surface of the room, displayed in meters squared (Required)

Tenants

The contacts that occupy the property (linked to res.partner).

  • Name: Name of the tenant (Required)
  • Phone: Phone number of the tenant (Required)
  • Email: Email of the tenant (Required)
  • Address: Address of the tenant (Street, City, Zip Code, State, Country) (Required)
  • Date of birth: Date on which the tenant was born (Optional)
  • Place of birth: Place in which the tenant was born (Optional)
  • Nationality: Nationality of the tenant (Optional)

Tenancies

The contracts binding tenants and a property over a specified period.

  • Property: A reference to the property rented (Required)
  • Tenants: A reference to the persons renting the property (Required)
  • Contract Name: Computed value (Property Name + First Tenant Name)
  • Start Date: The date on which the contract starts (Required)
  • Date End: The date on which the contract ends (Optional)
  • State: Draft, Active, Cancelled, or Terminated (Required). Default: draft.
  • Tenancy Lines: Reference to the lines of the contract.

Tenancy Lines

  • Product: The product linked to the line (e.g., Rent or Bills) (Required)
  • Quantity: Number of products sold (Required)
  • Price: Unit price of each product (Required)
  • Total Amount: Computed value (quantity * price)

Implementation Notes

  • Custom Models: Create appropriate models, views, and access rights for Properties, Tenancies, and Tenants.
  • Inheritance: Tenants are res.partner records. Add fields to the Python model and the existing partner form view.
  • Logic: Tenancies start in draft. Add a button to set to Active only if Date End > Date Start.
  • Automation: Create an ir.cron scheduled action to check daily if a tenancy has reached its end date.
  • Navigation: Create a new top-level menu item titled Realty Management.
